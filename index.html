Private. Ephemeral. Serverless. Nothing to censor, seize, or subpoena.

There is no relay to run or shut down. Every message and every download rides the onion-routed Veilid peer-to-peer network — designed so no single node ever learns both ends of a transfer.

What makes it different

No servers

Peer-to-peer over the public Veilid network. Nothing central to operate, block, or compel.

Hides both ends

Two stacked onion routes — one for the sender, one for the receiver. No relay in the path knows who is talking to whom.

Post-quantum

Everything daemonseed encrypts uses CNSA 2.0 — ML-KEM-1024, ML-DSA-87, AES-256-GCM. Content stays encrypted against a future quantum adversary. The transport's own crypto is classical for now.

Hard to block

There is no daemonseed server to blacklist — blocking it means blocking an entire public peer-to-peer network. We don't claim victory over determined national firewalls; we do remove every easy target.

Pure Rust

All daemonseed code — including its dedicated post-quantum crypto module — is memory-safe Rust. Only platform plumbing (storage, fonts, windowing) links C libraries.

Free & open

AGPL-3.0-or-later. Built clean — zero lines of Demonsaw code, informed only by its lineage.

No history

daemonseed keeps no chat history. Messages live only in memory — close the app and the conversation is gone, with nothing written to disk. This is not an ordinary chat app: your identity and circles persist (encrypted) so you can rejoin, but the messages themselves never do.

This is alpha software. It is early, it has rough edges, and it has not been through a security audit. It is for the curious and the technically brave — not yet for anyone whose safety depends on it. Verify every download (see the download page).
